# Overview

## Current Behavior

The public portfolio routes are functional but visually minimal. The public
shell uses a simple header, flat cards, and sparse layout. Admin screens have
usable CMS controls, but they do not yet share a polished design system.

## Target Behavior

Public portfolio routes adopt a polished developer-portfolio visual language
inspired by the referenced portfolio site: dark glass navigation, blue/cyan
gradient accents, hero CTA treatment, animated cards, richer section rhythm,
and responsive mobile navigation. Admin surfaces receive compatible tokens and
interaction polish without becoming a landing page.

## Affected Users

- Public portfolio visitor.
- CMS admin, indirectly through shared styling polish.

## Affected Product Docs

- `docs/TEST_MATRIX.md`

## Non-Goals

- Copying text, branding, images, or proprietary assets from the reference site.
- Backend API, database, auth, or media attachment changes.
- Completing all Phase 7 public content requirements.
